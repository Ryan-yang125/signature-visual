# Changelog

## 3.0.0 — Creative Search Engine

- Reconfigured creative effort for landmark, section, and refine scopes, with taxonomy-external landmark directions and distance-triggered branching.
- Added obvious-attractor quarantine, anti-literal candidates, four-axis pair admission, and a bundled replayable causal-offset generator to increase cross-run search entropy across substance, time, and interaction.
- Added coordinated offset cohorts with pairwise-distinct causal routes and spatial representation modes, preventing parallel runs from collapsing into the same renderer-friendly visual family.
- Added a convergence forecast and a two-axis causal cluster-break gate so landmark winners escape the artifact/material/time/interaction combination an independent run is most likely to repeat.
- Delayed Pattern Language until direction freeze and recast every card as a transformation operator, variation space, cheap-signal check, and failure diagnostic.
- Added candidate, source, and prior-output distance checks plus a compact schema-validated V3 design record.
- Added an executable design-record validator that replays creative offsets, verifies landmark direction origins, counts all six pairwise distance axes, and enforces the causal cluster-break evidence.
- Expanded temporal direction beyond one universal arc to held, one-shot, event-response, branching, inspection, irreversible, finite-state, scroll, ambient, accretive, and live structures.
- Added complete 1–5 evaluation anchors, score-5 evidence requirements, adversarial counterevidence, independent review, and separate artifact/release gates.
- Upgraded deterministic QA with capture, interaction, and production tiers; declarative runtime scenarios; capability coverage; derived thumbnail, blur, silhouette, and PNG contact-sheet evidence.
- Made supported capabilities prove their own action and semantic-assertion patterns, then committed each stored runtime description through an atomic render-and-describe boundary.
- Added ownership-marked staged artifact promotion so successful QA runs remove retired captures while preserving reviewer notes, failed runs restore the last successful evidence, and incomplete rollback data stays available for recovery.
- Resolved output targets through symbolic links and stabilized the virtual clock from a deterministic lead-in before every capture and runtime scenario.
- Added deterministic replay of all 12 public manifests, with exact CPU hashes and bounded same-host GPU pixel stability.
- Added executable Draft 2020-12 validation for the documented design record, documented production manifest, all public-case manifests, and every CI-generated QA result.
- Hardened all four runtime shells for zero-size pause, cancellation, window focus, runtime motion-preference changes, initialization rollback, fallback, context loss, and repeatable disposal/remount.
- Split starter pointer reporting into an immediate semantic `active` boolean and an eased numeric `strength`, then verified the shared cancellation contract in the browser harness.
- Upgraded all 12 public-case manifests to production-tier runtime scenarios and pinned the Cloudflare deployment tool version.
- Expanded the benchmark to nine scenarios across three scopes, obvious-attractor regression, static and animated work, renderer pressures, source study, mobile refinement, and output-distance control.
- Redesigned the public site as Edition 03 with interactive search entropy, temporal archetypes, adversarial selection, the post-freeze Critique Atlas, and V3 runtime evidence.
