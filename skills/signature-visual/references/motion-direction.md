# Motion Direction

Time is one design material among several. Use it when change communicates product behavior, causality, history, control, or state. A complete held composition is a valid temporal direction.

Choose the temporal structure after the visual direction freezes. Derive it from the governing rule instead of fitting every visual into one dramatic arc.

## Describe the temporal job

Answer only the questions the direction needs:

```text
TRIGGER       entry, data, pointer, scroll, command, schedule, or none
EVENTS        meaningful state changes in causal order; any useful count
HELD STATE    the frame that can remain on screen without explanation
END CONDITION completion, equilibrium, interruption, persistence, or open state
REVERSIBILITY what returns, what remains, and why
LATENCY       immediate, sampled, propagated, accumulated, or delayed
SECONDARY     quieter material behavior, if it adds evidence
```

Name visible changes such as topology, relation, position, density, material, annotation, camera, or history. Remove events that only fill time.

## Temporal archetypes

This list is open-ended. Select, combine, or author another structure when the subject requires it.

### Held or static

The composed state carries the full idea. Small material life may exist below the attention threshold, or the visual can remain still. Capture desktop, mobile, and reduced-motion frames plus relevant data variants.

### One-shot transformation

One consequential event changes the visual and reaches a designed end state. Define start, decisive change, and end through the subject's causality. Replay only when repetition helps understanding.

### Finite-state transition

The visual moves among named semantic states such as empty, acquiring, uncertain, resolved, or failed. Transitions may have different timing and do not need a universal peak.

### Event-response

Input or data causes acknowledgment, consequence, and a material-specific aftermath. Recovery is useful when the input can disturb the composition; persistent evidence is useful when history matters.

### Bifurcation / branching

One condition divides a route, population, surface, or state into visibly different futures. Define the branch cause, decision point, retained context, and whether branches can converge again. Branch count follows the subject.

### Inspection / reveal

A probe, focus, sample, scrub, aperture, or comparison exposes an existing hidden relation. Keep the revealed layer spatially tied to its source and define what remains visible after inspection ends.

### Irreversible event

A commit, cut, deposit, break, merge, signature, or threshold permanently changes the state. Design the before-state, event evidence, and aftermath. Reversal can require an explicit new event instead of automatic recovery.

### Scroll-linked sequence

Scroll exposes relations that need spatial continuity. Each segment carries a distinct narrative change. Endpoints form valid frames, and back-navigation remains deterministic.

### Bounded ambient cycle

A recurring system communicates an ongoing process. The cycle needs recognizable change and a stable seam. Perpetual drift with no consequence has no temporal job.

### Accumulation, accretion, or erosion

History changes the surface cumulatively. Define storage, saturation, pruning, reset, and the meaning of residue. A loop may be inappropriate.

### Live or data-paced

Cadence comes from real updates, samples, thresholds, or batches. Missing, stale, burst, and malformed states receive designed behavior. Decorative high frequency can misrepresent the data.

## Timing and causality

Describe easing through the material or system:

- tension can return elastically;
- fluid or particulate matter can settle with drag;
- instruments can acquire in steps or samples;
- emitted marks can depart ballistically and decay;
- dependencies can respond after causal delay;
- a cut, deposit, decision, or anomaly can leave persistent evidence.

Position, opacity, deformation, camera, and labels may use different response times. Synchronize only the events that need to read as one cause.

Use macro, meso, and micro timing when multiple scales genuinely exist. They remain optional descriptive tools.

## Interaction in time

Map input to a semantic role such as probe, pressure, focus, sample, route, resource, or commit. Define bounds before smoothing. Preserve a stable exit, cancellation, visibility-change, and unmount path.

An interaction can be immediate, damped, sampled, propagated, accumulated, or irreversible. Its aftermath should follow the governing rule. A pointer-following ornament adds no evidence by itself.

Skip interaction when observation communicates the idea more clearly.

## Reduced motion

Preserve the thesis and information hierarchy with the temporal strategy that fits:

- present the strongest held or resolved state;
- replace travel with immediate semantic state changes;
- lower amplitude while retaining causal material response;
- expose a small set of user-triggered states;
- show accumulated history as one readable composition.

Reduced motion can use a different composition and state path. Camera travel, large parallax, continuous drift, and unnecessary interpolation should disappear.

## Deterministic capture by archetype

Choose semantic checkpoints tailored to the selected archetype:

- **Held/static:** primary desktop, mobile, reduced motion, and relevant data states.
- **One-shot:** start, decisive transition, end, and replay seam when present.
- **Finite-state:** every named state plus risky transitions.
- **Event-response:** before input, acknowledgment, consequence, release or commit, aftermath.
- **Bifurcation/branching:** shared origin, branch cause, distinct paths, convergence or retained split.
- **Inspection/reveal:** covered state, approach, revealed relation, exit, retained evidence when applicable.
- **Irreversible event:** before, event, immediate consequence, persistent aftermath.
- **Scroll-linked:** each narrative boundary and both endpoints.
- **Ambient:** fixed seed at times showing materially different system behavior and the loop seam.
- **Accumulation/accretion/erosion:** initial, legible history, saturation, pruning or reset.
- **Live/data-paced:** empty, typical, burst, stale, malformed, and recovery where applicable.

Add pointer exit, touch cancellation, resize, reduced motion, and route teardown whenever the implementation owns those behaviors. Every capture should be explainable through the selected temporal archetype.

## Warning signs

Revise when:

- every project receives the same entry, build, peak, and recovery shape;
- motion changes only speed, color, or opacity while structure stays fixed;
- the resting frame needs several seconds before it feels intentional;
- micro-motion competes with primary copy;
- a loop seam flashes or resets causality;
- interaction strands the composition after exit;
- reduced motion removes the visual's meaning;
- time implies live data, precision, or causality the product does not provide.
